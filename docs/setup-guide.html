Step 1 of 13

Prerequisites

Check that your system has everything needed before we start. MedGuard AI runs on Windows, macOS, and Linux.

  • ✓
    Python 3.10+ — Check: python3 --version
  • Docker + Docker Compose — Check: docker --version
  • Git — Check: git --version
  • 8 GB RAM minimum (16 GB recommended for full stack)
  • Webcam or CCTV connected (for live detection)
  • 15 GB free disk space (for Docker images + ML models)
💡
No GPU required. All models run on CPU. A GPU (NVIDIA CUDA) will speed up inference 10-30×, but is completely optional.
OSStatusNotes
Ubuntu 20.04 / 22.04✓ RecommendedBest performance

Raspberry Pi 4Edge modeUse lightweight profile

Step 4 of 13 — NEW

Database Memory System

MedGuard AI has a built-in persistent memory layer using SQLite. It stores everything — detection events, patient profiles, chat history, agent decisions, and system health — automatically, with zero configuration.

⚡
Zero configuration needed. The memory DB starts automatically when the API starts. You just need to run one init command to verify it.

Step 11 of 13

Database Memory API Reference

Complete reference for querying and managing the persistent memory. Use these in any Python script or from the FastAPI endpoints.

①
Query detection events
▾
python
from backend.utils.memory_db import *

# Get last 50 events
events = get_detection_events(limit=50)

# Filter by type and severity
critical = get_detection_events(
    severity="critical",
    event_type="seizure",
    since_hours=24
)

# Get unresolved events
unresolved = get_detection_events(resolved=False)

# Resolve an event
resolve_event(event_id="uuid-here", resolved_by="Dr. Patel")

# 24-hour summary stats
stats = get_event_stats(hours=24)
print(stats)
# {'total_events': 12, 'unresolved': 3,
#  'by_type': {'fall': 5, 'seizure': 7},
#  'avg_confidence': 0.847}
②
Chat history management
▾
python
# Save a chat message
save_chat_message(
    session_id="sess_abc",
    role="user",
    content="What are the signs of a cardiac event?",
    patient_id="patient-uuid"
)

# Retrieve full history
history = get_chat_history("sess_abc", limit=20)
for msg in history:
    print(f"{msg['role']}: {msg['content'][:60]}")

# Get all sessions for a patient
sessions = get_all_sessions(patient_id="patient-uuid")

# Clear a session (e.g. on logout)
count = clear_session("sess_abc")
print(f"Deleted {count} messages")
③
Export + backup memory
▾
python
# Export entire DB to JSON
result = export_memory("./backup/export_2025.json")
print(result)
# {'path': './backup/export_2025.json',
#  'patients': 5, 'events': 347}

# Check DB health
info = get_db_info()
print(info)
# {'db_path': './data/medguard_memory.db',
#  'db_size_kb': 420.5,
#  'row_counts': {'patients': 5, 'detection_events': 347, ...}}
💡
The SQLite file at data/medguard_memory.db can be opened with any SQLite viewer like DB Browser for SQLite (free download).

Step 13 of 13

Troubleshooting

Common issues and how to fix them.

!
Camera not opening / black screen
▾
bash
# List available cameras
python3 -c "
import cv2
for i in range(5):
    cap = cv2.VideoCapture(i)
    if cap.isOpened():
        print(f'Camera {i}: AVAILABLE')
        cap.release()
"

# Fix: change CAMERA_SOURCES in .env to the correct index
# or grant camera permissions to Python/Terminal in System Settings
!
Database connection refused
▾
bash
# Check if postgres is running
docker ps | grep postgres

# If not running:
cd deployment && docker-compose up -d postgres

# The SQLite memory DB ALWAYS works without PostgreSQL
# Check it with:
python3 -c "from backend.utils.memory_db import get_db_info; print(get_db_info())"
!
Ollama / LLM not responding
▾
bash
# Check Ollama is running
curl http://localhost:11434/api/tags

# Restart Ollama container
docker restart $(docker ps -q --filter name=ollama)

# Pull model again if missing
docker exec $(docker ps -q --filter name=ollama) ollama pull llama3
!
Too many false detections
▾
.env
# Raise detection threshold (more conservative)
DETECTION_CONFIDENCE=0.80

# Increase cooldown between alerts (seconds)
ALERT_COOLDOWN_SECONDS=60

# Restart API after changing
